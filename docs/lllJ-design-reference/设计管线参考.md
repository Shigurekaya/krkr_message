# lllJ 设计管线参考（静态分析 · 学习用）

> 目的：理解**如何设计与插入** UI、立绘差分、SD、转场、表情等，以便用**自己的素材**做同类作品。  
> 来源：`E:\GAL\lllJ` 静态解包 + 全 CG 存档解码（**未启动游戏**）。  
> 生成：2026-07-12

## 1. 静态提取结论（重要）

| 内容 | 静态可读？ | 说明 |
|------|-----------|------|
| XP3 **文件索引**（hash/大小/数量） | ✅ | 见 `indexes/*.catalog.json` |
| **startup.tjs** 字节码 | ✅ | 见 `scripts/startup.tjs` |
| **存档 datasu/datasc**（资产 ID 登记） | ✅ | simple_crypt 可解码，见 `savedata/` |
| uipsd / image / fgimage **文件体**（PSD/TLG） | ❌ | 内容经 `limelight_lj.exe` 专有 filter 加密 |
| XP3 **文件名** | ❌ | 索引中多为单字符混淆名，无 eliF 明文表 |

**结论**：不启动游戏可还原「命名体系 + 系统配置 + 资源规模」，但**无法**直接得到 PSD/立绘图片。学习设计应依赖本目录的目录/命名/配置文档，素材本身需自行创作。

## 3. UI 设置界面 — 设计与插入

### 3.1 三层结构

```
[美术] Photoshop → uipsd.xp3 (PSD 图层)
[静态字] 烤入 TLG → image.xp3 (option_xx__pack.tlg)
[动态字] UTF-8 文案 → uitexts.toml（在 data.xp3 内，加密）
[逻辑]   TJS/UI 框架 → data.xp3（加密）
         psd.dll 读 psd:// 虚拟路径
         windowEx.dll 管窗口/Tab
         toml.dll 读 uitexts.toml
```

### 3.2 设置页状态机（从存档 autotrail 还原）

存档 `datasu.ksd` 中登记的 UI 轨迹名（**autotrail** = 自动播放/状态轨迹）：

| 轨迹名 | 推断用途 |
|--------|----------|
| `autotrail_title_option` | 标题 → 设置入口 |
| `autotrail_option_open` | 打开设置 |
| `autotrail_option_page` | Tab 翻页 |
| `autotrail_option_page_done` | 单页加载完成 |
| `autotrail_option_back` | 返回 |
| `autotrail_option_back_rclick` | 右键返回 |
| `autotrail_option_start_title` | 初始化本页 |
| `autotrail_option_wait` | 等待输入 |
| `autotrail_option_title` | 设置内回标题 |

**模仿要点**：设置不是单张图，而是 **TJS 状态机 + PSD Tab 页 + 热区**；每个状态对应一条 autotrail 可追踪。

### 3.3 设置变量（datasc.ksd 解码）

关键配置键见 `asset-catalog-from-save.json` → `config_keys`，例如：

- `lastOptionPage` — 上次打开的 Tab
- `option` — 设置总开关/结构
- `faceMode` / `chDefaultFace` — 表情显示逻辑
- `flowchartForceItemOpen` — 流程图项展开
- `touchUI_enabled` / `touchUI_opacity` — 触摸 UI
- `ShortCutKeyBinds` — 键位绑定
- `sysvoplay_option` / `sysvoplay_optpage` — 系统语音在设置页播放

## 4. 立绘 / 差分 / 表情 — 命名与插入

### 4.1 存放位置

- **普通立绘 + 差分**：`fgimage.xp3`（1232 文件，918 个 <50KB 多为差分小图）
- 运行时由 KAG/`[image]` + layer 管理；柚子社扩展 `yuzuex.dll`

### 4.2 事件 CG 命名（鉴赏登记）

从全 CG 存档解码得到 **2664** 个 `cg_*` ID，模式：

| 前缀 | 示例 | 含义 |
|------|------|------|
| `cg_EV` | `cg_EV101AA`, `cg_EV105MCI` | 事件 CG；末字母为差分/视角变体 |
| `cg_BGV` | `cg_BGV007_01_` | 画廊 BGV 条目 |
| `cg_SD` | `cg_SD401AB_RA01_YC04` | **SD 角色**（见下） |
| `cg_ATTENTION` | — | 系统/注意类 |

`EV` + 三位场景号 + 多位变体码：`AA/AB/BA/...` 表示同场景多构图/表情/差分。

### 4.3 SD 角色命名（569 条）

模式：`cg_SD{场景}{角色块}_{姿态}_{表情}`

示例：

```
cg_SD001A01          — 场景001 角色A 姿态01
cg_SD002_EA02_TB03   — 场景002 组合 EA02 姿态B03
cg_SD401AB_RA01_YC04 — 场景401 角色AB 姿态RA01 表情YC04
cg_SD404_Y02_M03     — 场景404 组Y02 表情M03
```

编码段含义（推断）：

- `SD` + 数字：场景/章节组
- `A/B/C` 或 `EA/EB`：角色或组合位
- `R*`：立绘姿势（Stand pose）
- `Y*` / `M*`：表情（Y=眼？M=嘴？需对照游戏）
- 末段数字：差分序号

**模仿要点**：先定 **ID 表**（如 Excel/JSON），脚本用 ID 加载 `fgimage` 内对应 TLG/图层，而非随意文件名。

### 4.4 表情 / H 场景脸部

`face_parts` 来自存档：

```
part_109_sel2_FACE
part_115_sel1_FACE
...
hselectMOUTHFACE
```

表示 **按剧本 part 编号 + 选择肢 + FACE 层** 切换脸部差分，与 `faceMode` 配置联动。

## 5. 转场设计

### 5.1 引擎 API（exe 字符串）

- `beginTransition` — 转场开始
- `onTransitionCompleted` / `onFadeCompleted` — 完成回调
- `Scrossfade` — 交叉淡入淡出
- `GlitchEffect.dll` — 故障风转场插件

### 5.2 本作登记的转场轨迹

```
autotrail_custom_trans_glitch
```

说明至少有一种 **glitch 自定义转场** 与 UI/演出系统挂钩（非简单 fade）。

**模仿要点**：转场 = **专用 Layer + 插件效果 + TJS 回调**；在 GAL 可用 `[trans]` / 自定义 TJS TransitionHandler 仿流程，不必抄素材。

## 8. 在 GAL 框架中「等价模仿」路线

| lllJ 做法 | 你可做的等价物 |
|-----------|----------------|
| uipsd + psd.dll | 外部 PS 出 PNG + 坐标 JSON |
| uitexts.toml | `Config.tjs` 或自研 `uitexts.json` |
| fgimage ID 表 | 自研 `characters.json` + `[image storage=...]` |
| autotrail 状态机 | 自研 `SettingsUI.tjs` 状态枚举 |
| SD 命名 cg_SD* | 自研 `sd_sprites.yaml` |
| glitch 转场 | Glitch 着色器或渐变 Layer（可选） |
